EventSourcing

Capture all changes to an application state as a sequence of events.

Martin Fowler

What is EventSourcing?

  • a kind of event driven architecture
  • sequence of events instead of a database record
  • saving the whole history instead of a snapshot
  • always inside one service
  • EventSourcing should only be used inside a service and not between them.
  • It should not be used for communication, only for saving state.

When to use EventSourcing?

  • When you need history or audit logging for example in finance
